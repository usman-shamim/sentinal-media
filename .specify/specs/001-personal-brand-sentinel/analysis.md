## 2. Spec vs Clarifications (3 GAPS — needs fixing)

### Gap A: Job Queue still referenced in spec edge cases

**Location**: `spec.md:114`
**Current text**:
```
- **Postiz rate limited (30 req/hour)**: Sentinel queues excess requests in the Job Queue table and retries on the next heartbeat cycle.
```

**Problem**: Clarify.md §2 explicitly deferred Job Queue to v1.1. Decision was "return error, let upstream retry." The edge case still describes the deferred Job Queue approach.

**Fix**: Replace with Postiz 429 retry-after pattern.

### Gap B: Missing FR — Postiz 429 rate-limit handling

**Location**: `spec.md` — nowhere in FRs
**Problem**: Clarify.md §2 said "Add FR to handle Postiz 429 rate-limit errors with retry-after." No such FR exists.

**Fix**: Add FR-018.

### Gap C: Missing FR — Auto-expire pending approvals

**Location**: `spec.md` — nowhere in FRs
**Problem**: Clarify.md §3 decided "auto-expire after deadline passes." FR-005 covers checking `is_expired()` before processing, but there's no requirement to auto-expire approvals already sitting in the queue when their deadline passes. `/cleanup` endpoint exists (T018) but isn't required in the FRs.

**Fix**: Add FR-019, or extend FR-005 to cover background expiry sweep.
